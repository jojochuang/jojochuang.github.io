Release 2.0.0 available

Apache Ozone 2.0.0 adds 1708 new features, improvements and bug fixes on top of Ozone 1.4.

The format is based on Keep a Changelog,
and this project adheres to Semantic Versioning.

[2.0.0] - 2025-04-04

Added

  • Move S3 Gateway web admin to separate port (HDDS-7307) - S3 Gateway endpoints for static content and admin purposes (/prom, /logs, etc.) are now served on a separate port (default: 19878). New configuration keys are under ozone.s3g.webadmin.
  • Symmetric Keys for Delegation Tokens (HDDS-8829) - Applied symmetric keys for delegation tokens.
  • Add CapacityPipelineChoosePolicy considering datanode storage space (HDDS-9345).
  • Create APIs to fetch single datanode related information (HDDS-9648).
  • Extract k8s definitions for HttpFS and Recon from getting-started example (HDDS-11845).

Changed

  • Enable container scanner by default (HDDS-10485).
  • Improve work with arrays (HDDS-11544).
  • Replace ConcurrentHashMap with HashMap protected by ReadWriteLock in NodeStateMap (HDDS-10830).
  • Replace ConcurrentHashMap with HashMap in PipelineStateMap (HDDS-10971).
  • Read chunk files using netty ChunkedNioFile (HDDS-7188).
  • Support passing Kerberos credentials in GrpcOmTransport (HDDS-11041).
  • Pipeline failure should trigger heartbeat immediately (HDDS-9823).
  • Recon datanode ‘Last Heartbeat’ should print relative values (HDDS-9933).
  • Propagate group remove to other datanodes during pipeline close (HDDS-9959).
  • Improve the response for the Deleted Directories Insight Endpoint (HDDS-10241).
  • Recon - Improve performance of processing IncrementalContainerReport requests (HDDS-9883).
  • Revert workaround added by HDDS-8715 to preserve thread name (HDDS-12470).
  • Extract method for building OMRequest in TrashOzoneFileSystem (HDDS-6796).
  • Use md5 hash of multipart object part’s content as ETag (HDDS-9680).
  • Checksum to support direct buffers (HDDS-10288).
  • Wrapper for set/get Standalone, Ratis and Rest Ports in DatanodeDetails (HDDS-117).
  • Managed objects should not override finalize() (HDDS-9528).
  • [UI] Migrate Recon from react-script to Vite (HDDS-11017).

Removed

  • Remove LegacyReplicationManager (HDDS-11759).
  • [Recon] Fix Typescript configs for Ozone Recon (HDDS-11198).

Fixed

  • TriggerDBSyncEndpoint admin only API handling in Recon (HDDS-11436).
  • Disable REST endpoint for S3 secret manipulation by username (HDDS-11040).
  • Improve log for container not found in CloseContainerCommandHandler (HDDS-9958).
  • Update Container for named RwLock operations (HDDS-10072).
  • Add Integration Tests for Failing Datanode Decommission and Maintenance Early (HDDS-10464).
  • Container Report admin command displays incorrect value after SCM restart (HDDS-10272).
  • Fix pagination on the OM DB Insights page in Recon (HDDS-10282).
  • IllegalArgumentException: containerSize Negative (HDDS-10293).

Security

  • Disable REST endpoint for S3 secret manipulation by username (HDDS-11040).
  • Move S3 Gateway web admin to separate port (HDDS-7307).

Dependency Updates

  • Bump vite to 4.5.5 (HDDS-11467).
  • Upgrade jgrapht to 1.4.0 (HDDS-10503).
  • Bump follow-redirects to 1.15.6 (HDDS-10526).
  • Bump axios to 0.28.0 (HDDS-10669).
  • Bump es5-ext to 0.10.64 (HDDS-10673).
  • Bump ip to 1.1.9 (HDDS-10674).
  • Bump browserify-sign to 4.2.3 (HDDS-10676).
  • Bump plotly.js to 2.25.2 (HDDS-10677).
  • Bump express to 4.21.0 (HDDS-11460).

For more details, check out Apache Ozone 2.0.0 JIRA list

This is a generally available (GA) release. It represents a point of API stability and quality that we consider production-ready.
